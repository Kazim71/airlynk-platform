# Runtime Validation & Verification

AirLynk mandates strict runtime verification via the browser to ensure the platform is fundamentally healthy. 

## Checklist for Validation

1. **Docker Containers**
   - Run `docker compose ps` to ensure `api`, `celery`, `redis`, `rabbitmq`, `postgres`, `grafana`, `prometheus`, `tempo`, and `loki` are running.

2. **Swagger UI**
   - Navigate to `http://localhost:8000/docs`.
   - Ensure the UI loads and displays all endpoints without Content Security Policy (CSP) errors.

3. **Prometheus Targets**
   - Navigate to `http://localhost:9090/targets`.
   - Ensure `airlynk-api` target is `UP`.

4. **Grafana Dashboards**
   - Navigate to `http://localhost:3000`.
   - Go to Dashboards -> General.
   - Verify all 8 provisioned dashboards are present and loading data.

5. **Tempo Tracing**
   - In Grafana, go to Explore -> Select Tempo as data source.
   - Search for traces generated by triggering an API endpoint.

6. **Loki Logging**
   - In Grafana, go to Explore -> Select Loki.
   - Run query `{job="api"}` to see structured JSON logs with injected `trace_id`.
